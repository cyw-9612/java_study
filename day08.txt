
1.多态
	关于java语法中的多态语法机制：
 		1.Animal,Cat,Bird三个类之间的关系
 			Cat继承Animal
 			Bird继承Animal
 			Cat和Bird之间没有继承关系
 
 		2.面向对象的三大特征：封住，继承，多态

  		3.关于多态涉及的几个概念：
  			*向上转型(upcasting)
  				子类型-->父类型
  				又被称为：自动类型转换
  			
  			*向下转型
  				父类型-->子类型
 				又被称为：强制类型转换。【需要加强制类型转换符】
  
  			*需要记忆：无论是向上还是向下，两者之间必须有继承关系

  	实例分析：
  	 1.一下程序可以通过编译阶段
		  
		  2.先分析编译阶段，再分析运行阶段，编译无法通过，程序无法运行
		  
		  3.编译阶段编译器检查a1这个引用的类型为Animal，而Animal.class字节码文件有move方法，所以编译通过了。
		  这个过程称为静态绑定，编译阶段绑定。
		  只有静态绑定成功，才有后续的程序运行。
		  
		  4.在运行阶段，JVM堆内存中创建的是Cat对象，所以运行中调用的是Cat对象的move();方法。
		  
		  5.父类型引用指向子类型对象，这种机制再编译阶段绑定和运行阶段绑定两种不同形态。
		  这种机制可以称为一种多态语法机制。

		  a2是Animal类型，可以向下转换为Cat类型
		  但是a2在堆内存存储的是一个Bird对象，Bird对象无法转换为Cat类型
		  类型之间不存在任何继承关系，此时出现著名异常：
		  		java.lang.ClassCastException
		  		类型转换异常，只会在向下转型时发生
	
	2.对于向下转型的处理方法 
		//Cat c2 = (Cat)a2;
		
		
		  1.以上异常只会在强制类型转换时发生，向下转型存在“隐患”（编译过，运行出错）
		  
		  2.向上转型就不一样，编译过，运行过
		  
		  3.如何避免向下转型出现.ClassCastException呢？
		  		使用instanceof运算符可以避免出现以上异常
		  
		  4.instanceof运算符怎么用？
		  		4.1语法格式
		  			(引用 instanceof 数据类型名)
		  
		  		4.2以上运算符执行结果时布尔类型的。
		  
		  		4.3关于运算结果true/false：
		  			假设(a instanceof Animal)
		  			*true表示：
		  				a这个引用指向对象是Animal类型
		  			*false表示：
		  				a这个引用指向对象不是Animal类型
		  
		  5.在强制类转换的时候，建议采用instanceof进行判断

	3.多态在实际开发中的作用
		*降低程序的耦合度，提高程序的扩展力
		*能使用多态尽量使用多态
		*父类型引用指向子类型对象

		核心：面向抽象对象编程，尽量不要面向具体对象

2.关于Java语言中fianl关键字：
	1.final是一个关键字，表示最终的，不可变的。
	2.final修饰的类无法被继承
	3.final修饰的方法无法被覆盖
	4.final修饰的变量一旦被赋值，无法再修改
	5.final修饰的实例变量
		由于实力变量有默认值 + final修饰的变量赋值后无法修复
		在对实例变量使用final修饰后，必须手动赋值【或者在构造方法执行时赋值】不能采用默认值
	6.常用固定变量定义格式
		public static final 类型 变量名 = 值;

3.关于Java语言中的包机制：package，import关键字
	1.包又称为package，java引入package这种语法机制是为了方便程序管理
	不同功能的类可以分门别类的放入不同的软件包中

	2.如何定义package？
		-在源程序第一行写package语句
		-package只能编写一个语句
		-语法结构：
			package 报名;

	3.包名的命名规范：
		公司域名倒序 + 项目名 + 模块名 + 功能名；
		采用这种方法重名纪律低，因为公司的域名全球唯一
		例如：
			com.baidu.oa,user.servise;
			org.apache.tomcat.core;

	4.包名全部小写，包名也是标识符，必须遵守标识符命名规则

	5.一个包对应一个目录

	6.使用package机制后，编译运行的方法
		-使用package机制后，类名需要加上包名
		-编译：javac java源文件路径（在硬盘上生成classwenjian）
		-手动方式创建文件夹，jiang .class 文件放在目录下
		-在源目录运行

		-方法二：（编译加运行）
			
			*编译：
				javac -b 编译后存放的路劲 java源文件路径
			
			*；例如：将F:\\Hello.java文件编译后放到c:\目录下
				javac -d c:\ F:\Hello.java
			
			*javac -d .*.java
				将当前路径中所有java文件编译后放到当前目录下

			*运行：
				保证DOS处于com所在的目录，

		7.import语句用来导入其他类，同一个包下的不用导入
			不同一个包的需要手动导入

			-import语法格式：
				import 类名; 【完整的：包名.类名】
				import 包名.*;

			-import语句需要编写到package语句之下，class语句之上

			-什么时候需要import
				要用的类不在在java.lang.*中，且不在一个包中

			-MyEclipse导入类的快捷键：
			 	Ctrl + Shift + O ；

4.访问控制权限修饰符：
	4.1访问控制权限修饰符用来控制元素的访问范围 

	4.2访问控制权限修饰符包括：
		public		公开的，可以在任何位置访问
		protected	受保护的，同包以及子类可以访问
		缺省		只能同包访问
		private		私有的，只能来本类中访问

	4.3访问控制权限修饰符可以修饰类，变量，方法。。。

	4.4当莫格数据只希望子类使用的时候，使用protected

	4.5修饰符的范围：
		private < 缺省 < protected < public 

	4.6类只能采用public 和 缺省修饰，内部类除外。
